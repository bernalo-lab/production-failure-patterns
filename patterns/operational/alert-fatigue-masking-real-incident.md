# Pattern: Alert Fatigue Masking Real Incident

## Description

Alert Fatigue occurs when engineers receive too many low-value alerts, causing them to ignore notifications or delay responses.

When a real incident occurs, it may be overlooked or dismissed because responders are accustomed to frequent false alarms.

---

# Typical Symptoms

**Examples**:

repeated alerts ignored
delayed incident response
alerts acknowledged but not investigated
important alerts lost among noise
slow escalation

---

# Common Misleading Signals

**Examples**:

looks like minor recurring issue
appears like known alert noise
seems like monitoring glitch
engineers dismiss alerts automatically

---

# Likely Root Causes

**Examples**:

poorly tuned alert thresholds
excessive alert rules
lack of alert prioritisation
duplicate alerts across systems
missing alert correlation

---

# First Checks

**Examples**:

review alert history
inspect alert severity configuration
check alert correlation
review responder behaviour
examine alert escalation policies

---

# Useful Signals

**Examples**:

alert frequency metrics
acknowledgement timelines
incident response logs
monitoring rule configuration
historical incident data

---

# Prevention

**Examples**:

alert noise reduction
alert prioritisation
monitoring rule reviews
alert correlation systems
periodic alert audits
