# Pattern: Incident Misclassification

## Description

Incident Misclassification occurs when an incident is incorrectly categorised during triage.

Incorrect classification can delay escalation, route the incident to the wrong team, or cause responders to pursue incorrect investigation paths.

---

# Typical Symptoms

**Examples**:

incident routed to wrong team
delayed response escalation
incorrect severity assigned
responders investigating unrelated systems
prolonged incident diagnosis

---

# Common Misleading Signals

**Examples**:

symptoms resemble known issue type
monitoring alerts suggest incorrect root cause
dashboards point to unrelated service
early diagnosis assumptions

---

# Likely Root Causes

**Examples**:

incomplete incident data
lack of structured triage process
poor alert descriptions
unclear ownership boundaries
rushed incident response

---

# First Checks

**Examples**:

validate incident classification
review affected services
inspect dependency graph
confirm incident severity
check alert context

---

# Useful Signals

**Examples**:

alert metadata
service dependency mapping
incident timeline
error logs
system health metrics

---

# Prevention

**Examples**:

structured incident triage frameworks
clear ownership documentation
improved alert descriptions
incident training exercises
automated incident classification
